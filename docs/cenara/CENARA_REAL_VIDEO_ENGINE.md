# Cenara real video engine

Cenara now uses a thin adapter in `webui/Main.py` instead of a simulated UI flow. The premium creator collects briefing fields, maps them into the existing MoneyPrinterTurbo `VideoParams` object, and calls `app.services.task.start` through `cenara_trigger_real_generation`.

## Real generation flow

1. The private operator gate validates `GX1_ACCESS_TOKEN` before the console loads.
2. `cenara_build_generation_payload` maps the premium fields to engine fields:
   - `tema` -> `video_subject`.
   - `roteiro_manual` -> `video_script`.
   - `fonte_video` -> `video_source`.
   - `voz_tts` -> `voice_name`.
   - `palavras_chave` -> `video_terms`.
   - `formato` -> `video_aspect`.
   - `duracao` -> `video_clip_duration`.
   - `nicho`, `publico`, `promessa`, and `cta` enrich the subject/script prompt.
3. `cenara_validate_generation_payload` blocks missing subject/script, missing video provider, or missing voice/audio before the backend is called.
4. `cenara_trigger_real_generation` calls the original MoneyPrinterTurbo task service with a real UUID task id.
5. `cenara_find_latest_mp4` searches the task/output storage for a non-empty `.mp4` generated by the engine.
6. Preview and download are only shown when a real MP4 exists.

## Required providers

- LLM provider: needed when the script must be generated from a subject.
- Video source: Pexels, Pixabay, Coverr, or local media.
- Voice/TTS: an available voice provider or explicit no-voice/custom-audio workflow.
- Rendering: FFmpeg must be available for video assembly.
- Subtitles: ImageMagick is detected when available and shown as optional/readiness information.

Secrets are never printed in the provider center or generation logs. Key input fields remain password fields and empty fields preserve existing configuration.

## Fallback modes

- If `tema` is empty and `roteiro_manual` is present, Cenara derives the subject from the first sentence and sends the manual script to the engine.
- If `roteiro_manual` is empty and `tema` is present, Cenara sends `video_subject` and lets the existing script generation path create the script.
- If both are empty, generation is blocked with a PT-BR error before any backend call.
- Local media can be used when provider keys are not available, provided uploaded/local materials exist in the standard flow.

## Preview and download

`cenara_render_real_preview` renders the MP4 with `st.video`, shows file size and generation time, and exposes a `st.download_button` for the exact MP4. `cenara_render_recent_videos` lists recent MP4s from storage without exposing absolute server paths.

## No fake success

Cenara must not show success unless a non-empty MP4 is found after the MoneyPrinterTurbo task finishes. If the task returns without a file, the UI reports that no MP4 was found and asks the operator to inspect the output directory.
